2016.11.11 - Stalled!
Занимался модулем пересылки уведомлений. Успеха не достиг. Обнаружились следующие проблемы:

* Если десериализация создает Notification, то теряются сведения об источнике (packageName)
* Если десериализация создает StatusBarNotification, то нужен API level 18 не только для отправителя, но и для получателя
* Или надо делать свой аналог StatusBarNotification?

* Мой RemoteNotificationManager не может отслеживать уведомления отмененные пользователем, что есть источник утечки
* Можно построить механизм его взаимодействия с Activity, но это неприятные зависимости
* Можно использовать getActiveNotifications(), но это API level 23
* Можно запускать NotificationListenerService на получателе, но это API level 18 и плохо укладывается в цель
* Можно сделать простой сервис, который постит уведомления:
  * RemoteNotificationManager отправляет этому сервису уведомления используя bind
  * В уведомления добавлен deleteIntent информирующий сервис
